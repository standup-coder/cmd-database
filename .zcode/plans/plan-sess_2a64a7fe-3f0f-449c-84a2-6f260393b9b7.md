# 重构方案：收敛 CLI 到 tools/cmd，根目录扁平化为中文领域 md 库

## 目标与约束
- 根目录扁平化为「中文领域文件夹 / 命令 md」**严格二层**结构（约 100 个领域文件夹）。
- CLI（cmd/ internal/ pkg/ go.* embed.go 等）整体收敛到 `tools/cmd/` 并**保持可用**。
- 内容源用 `llm-wiki/01-Commands/*.md` + `best-practices/bp-*.md`（带 frontmatter、双向链接）。
- 命名：文件夹与文件名均用**中文**（领域）+ 命令原名。
- **先迁移再删旧**，分批 `git mv`，每步可验证、可回退；不直接 `rm` 重建。

## 关键技术依据（已验证）
1. **`cmd_category` frontmatter 不可靠**：1112 个命令页中仅 760 个用干净的 `领域/子领域` 斜杠格式，**352 个**是英文裸域（如 `"Network Tools"`/`"Database"`/`"Operating System"`）。→ **主映射源用 `data/*.yaml` 的 `category` 字段**（干净中文），frontmatter 与翻译表作兜底。
2. **双向链接移动后不坏**：2538 条 `[[wikilink]]`，0 条路径链接，全是按 basename 解析的 `[[命令名]]` 或 `[[领域-MOC|别名]]`。只要文件名不变，迁移到子文件夹不影响 Obsidian 链接。
3. **best-practices**：717 个 `bp-*.md`，711 个可匹配命令页，6 个孤儿（命名空格/标点差异）。→ 同领域文件夹内放 `bp-<命令>.md`，孤儿保留到最近领域。
4. **Go CLI 可保活**：唯一硬绑是 `embed.go` 的 `//go:embed all:data`；其余均由 `dataDir` 参数化（验证器默认 `./data`）。→ `data/*.yaml` 移到 `tools/cmd/data/`，同步改 embed 路径、验证器默认值、测试相对路径即可。
5. **`scripts/` 已损坏**：全部脚本硬编码旧绝对路径 `.../standup-coder/cmd4coder`（重命名前），现仓库是 `cmd-database`。→ 收敛到 `tools/cmd/scripts/` 时一并把硬编码改为相对仓库根。

## 执行步骤

### 阶段 0：建领域映射表（单一真相源）
- 写一个 Python 脚本 `tools/cmd/scripts/build_domain_map.py`，读取 `data/*.yaml` 每个命令的 `category`（如 `AI基础设施/大模型训练` → 领域 `大模型训练`），产出 `tools/cmd/scripts/domain_map.json`：
  ```
  { "<命令名 slug>": { "domain_cn": "大模型训练", "category": "AI基础设施/大模型训练", "source_yaml": "ai/llm-training.yaml" } }
  ```
- 352 个 frontmatter-only 命令用「frontmatter → 翻译表」兜底（`Network Tools`→`HTTP工具`等，从现有 MOC 名对齐）。翻译表也存进 `domain_map.json`。
- 输出报告：列出每个命令的目标文件夹，**先人工抽查**（deepspeed/arthas/kubectl/npm/mysql/ab 等代表）再继续。

### 阶段 1：收敛 CLI 到 `tools/cmd/`（保活）
用 `git mv`（保留历史）：
```
cmd/        → tools/cmd/
internal/   → tools/cmd/internal/
pkg/        → tools/cmd/pkg/
embed.go    → tools/cmd/embed.go
go.mod      → tools/cmd/go.mod
go.sum      → tools/cmd/go.sum
Makefile    → tools/cmd/Makefile
data/       → tools/cmd/data/        # YAML 单一数据源，保 embed 绑定
scripts/    → tools/cmd/scripts/     # 含上面 build_domain_map.py
bin/        → tools/cmd/bin/（gitignored，可不迁）
test/       → tools/cmd/test/
```
然后修引用让 CLI 仍能 build/test：
- `tools/cmd/embed.go`：`//go:embed all:data`（相对 `tools/cmd/` 仍成立）。
- `tools/cmd/cmd/validator/main.go`：默认 `-d ./data` 改为相对新结构（`tools/cmd/data`）或 `../../tools/cmd/data`。
- `tools/cmd/**/*_test.go`：相对路径 `"../../data"` → 调整。
- `tools/cmd/scripts/*.py`：所有 `Path("/Users/.../cmd4coder")` → `Path(__file__).resolve().parents[N]`（相对脚本定位仓库根）。
- **验证**：`cd tools/cmd && go build ./cmd/cli && go test ./...` 全绿才进下一阶段。

### 阶段 2：根目录建领域 md 库（从 llm-wiki 迁移）
写 `tools/cmd/scripts/migrate_wiki_to_root.py`，对每个 `llm-wiki/01-Commands/<cmd>.md`：
1. 查 `domain_map.json` 得 `domain_cn`。
2. `mkdir -p <根>/<domain_cn>`，`git mv llm-wiki/01-Commands/<cmd>.md <根>/<domain_cn>/<cmd>.md`。
3. 对应的 `llm-wiki/best-practices/bp-<cmd>.md`（若存在）→ `git mv` 到同一 `<domain_cn>/bp-<cmd>.md`。
4. 6 个孤儿 bp → 归入其命令所在领域（按 `cmd_name` 前缀如 `kubectl-*` → `Kubernetes命令`）。
5. 跳过 `01-Commands/index.md`（索引类，单独处理）。

**预期结果**：根目录约 100 个中文领域文件夹，每个内含若干 `<命令>.md` + `bp-<命令>.md`，无嵌套子目录。

### 阶段 3：根目录导航与辅助文件
- 新建根 `INDEX.md`：列出所有领域文件夹（链接），取代旧 INDEX.md。
- 每个领域文件夹放一个 `_MOC.md`（从 `llm-wiki/00-Maps/<领域>-MOC.md` 迁来并修正链接），作为该领域入口。
- `llm-wiki/02-Scenes`、`03-Concepts`、`04-FAQs`、`05-Troubleshooting` → 根目录 `场景/`、`概念/`、`FAQ/`、`排障/`（保持二层）。
- 顶层 `README.md` 重写：说明新结构、tools/cmd 用法、领域索引。

### 阶段 4：清理旧目录（验证后）
**仅当阶段 1-3 全部验证通过**，删除已迁空的旧目录：
- `llm-wiki/`（`.obsidian/` 配置可选保留到根 `.obsidian/` 以便继续用 Obsidian 浏览）
- 根 `COMMANDS.md`（由 data 重新生成到 `tools/cmd/` 或根 INDEX 取代）
- 旧根 `cmd/ internal/ pkg/ data/ scripts/ embed.go go.* Makefile`（已在阶段 1 git mv 走，此步只清残留）
- 更新 `.gitignore`：`bin/`→`tools/cmd/bin/` 等。

### 阶段 5：验证
- `git status` 确认无遗漏、无未跟踪残留。
- 根目录结构自检：所有领域文件夹内均为 `.md`、无子目录（二层结构断言）。
- 链接自检脚本：扫描所有 `[[wikilink]]`，确认 target 文件名在新结构中存在（验证阶段 2 结论）。
- CLI build/test 再次全绿。
- `cd tools/cmd && go run ./cmd/validator -d ./data -v` 通过。

## 不在本次范围
- 不重写命令 md 内容（仅迁移、必要时修链接/路径）。
- 不改 frontmatter schema。
- 不动 `.agents/skills/`（与 wiki 运维 skill 解耦，可单独评估）。
- `bin/cmd4coder` 旧二进制（gitignored）不迁。

## 风险与回退
- 全程 `git mv`，任何阶段失败可 `git reset` 回退，无内容丢失风险。
- 阶段 0 产出映射表后**先暂停给你抽查**，确认领域归类正确再执行阶段 2 的大批量迁移。
- 阶段 1 CLI 保活独立验证，与阶段 2/3 解耦。
